# 勤怠管理アプリ

## 概要
Laravelを用いて開発した勤怠管理アプリです。
一般ユーザーによる出勤・退勤・休憩の打刻、勤怠修正申請、管理者による勤怠修正・承認機能を提供します。

## 環境構築

1. リポジトリをクローン
```bash
git clone git@github.com:urbexsaku/kintai-app.git
```
2. Docker Desktopを起動

3. プロジェクト直下で、以下のコマンドを実行する
```bash
make init
```

※ make initでは以下を実行します。
- Dockerコンテナのビルド・起動
- Composerパッケージのインストール
- 環境変数ファイルの作成
- アプリケーションキーの生成
- storageおよびbootstrap/cacheの権限設定

4. make init 実行後、以下を実行してください。

```bash
make fresh
```

※ データベースの初期化およびテストデータ投入を実行します

### テストについて補足

- ID 2「メールアドレスが未入力の場合、バリデーションメッセージが表示される」「パスワードが未入力の場合、バリデーションメッセージが表示される」についてはログインバリデーション単体の検証であるため、ユーザー登録処理は不要としてテストを実装
- ID 4「日時取得機能」については日時表示がJavaScriptによるクライアントサイド描画のため、PHPUnit（サーバーサイドテスト）の対象外として扱い、テストは実施しない
- ID 16「メール認証誘導画面で「認証はこちらから」ボタンを押下するとメール認証サイトに遷移する」については、外部サイトへの遷移機能のため、本機能はPHPUnitのテスト対象外とする

## 追加実装機能

以下は、開発要件に記載のない追加実装機能です。

### 会員登録 / ログイン
- メール認証誘導画面で認証メール再送後、「認証メールを再送信しました」を表示
- 登録済みメールアドレスで会員登録しようとした場合、「このメールアドレスは既に登録されています」と表示
- 管理者が一般ログイン、あるいは一般ユーザーが管理者ログインしようとした場合、「ログイン情報が登録されていません」を表示

### 勤怠打刻
- 同日に2度目の出勤打刻をしようとした場合、「本日は既に出勤済みです」と表示

### 勤怠一覧
- 勤怠データが存在しない日付の詳細画面は未実装（要件外仕様）として扱い、詳細ボタンによる遷移は行わない
- CSV出力は勤怠データが存在する日付のみを対象として出力する

### 勤怠修正申請
- 管理者による勤怠修正後は、「勤怠データを更新しました」というメッセージを表示
- 一般ユーザーは、一度修正申請が承認された後は、同一日付の勤怠に対して再度修正申請を行うことができる
- 休憩時間および出退勤時間の入力バリデーションを強化し、以下の不正入力を許可しない
  - 出勤時間が未入力の場合：「出勤時間を入力してください」
  - 退勤時間が未入力の場合：「退勤時間を入力してください」
  - 休憩開始時刻が退勤時刻より後の場合：「休憩時間が不適切な値です」
  - 休憩終了時刻が退勤時刻より後の場合：「休憩時間もしくは退勤時間が不適切な値です」
- 休憩時間の入力において、開始・終了の組み合わせが不正な場合は複数のバリデーションエラーが同時に表示される場合があるが、要件上その挙動を許容している

### 修正申請承認
- 管理者が勤怠承認後、「承認しました」というメッセージを表示
